# 贡献者行为准则

## 我们的承诺

为了营造一个开放和欢迎的环境，我们作为贡献者和维护者承诺，无论年龄、体型、残疾、种族、性别认同和表达、经验水平、教育背景、社会经济地位、国籍、个人形象、种族、宗教或性取向如何，参与我们项目和社区的每个人都能享受无骚扰的体验。

## 我们的标准

有助于创造积极环境的行为包括：

### ✅ 积极行为

* **使用友好和包容的语言** - 尊重不同的观点和经验
* **优雅地接受建设性批评** - 将反馈视为改进的机会
* **关注对社区最有利的事情** - 优先考虑集体利益
* **对其他社区成员表现出同理心** - 理解和支持他人
* **保持专业和礼貌** - 在所有互动中保持尊重
* **分享知识和经验** - 帮助他人学习和成长
* **承认他人的贡献** - 给予应有的认可和感谢
* **建设性地解决冲突** - 寻求双赢的解决方案

### ❌ 不当行为

不可接受的行为包括：

* **使用性别化的语言或图像** - 以及不受欢迎的性关注或性骚扰
* **恶意评论、人身攻击或政治攻击** - 包括侮辱性/贬损性评论
* **公开或私下骚扰** - 任何形式的骚扰都不被容忍
* **未经明确许可发布他人的私人信息** - 如物理地址或电子邮件地址
* **其他在专业环境中被认为不当的行为** - 任何可能造成不适的行为
* **故意传播错误信息** - 误导或欺骗社区成员
* **破坏性行为** - 故意干扰项目进展或社区和谐
* **歧视性言论** - 基于任何个人特征的歧视

## 技术行为准则

### 代码质量标准

* **编写清晰、可维护的代码** - 遵循项目编码规范
* **提供充分的文档** - 确保代码易于理解和使用
* **编写有意义的提交信息** - 清楚描述更改内容和原因
* **进行充分的测试** - 确保代码质量和稳定性
* **及时响应代码审查** - 积极参与审查过程
* **尊重现有架构** - 在进行重大更改前进行讨论

### 协作标准

* **使用清晰的问题描述** - 提供足够的上下文和重现步骤
* **保持讨论的专业性** - 专注于技术问题而非个人情感
* **接受不同的技术观点** - 开放地讨论各种解决方案
* **及时更新状态** - 让团队了解您的工作进展
* **寻求帮助时要具体** - 提供详细信息以便他人协助

## 执行

### 举报流程

如果您遇到滥用、骚扰或其他不可接受的行为，请通过以下方式举报：

#### 🚨 紧急情况
* **严重违规行为** - 立即发送邮件至 pikachu237325@163.com
* **骚扰或威胁** - 同时可在GitHub上私信项目维护者

#### 📝 一般举报
1. **收集证据** - 截图、链接、时间等相关信息
2. **详细描述** - 说明事件经过和影响
3. **发送举报** - 通过邮件或GitHub私信方式
4. **等待处理** - 我们将在48小时内回复

### 处理程序

项目维护者有责任和权利：

#### 🔍 调查阶段
* **及时响应** - 在48小时内确认收到举报
* **公正调查** - 收集所有相关信息和证据
* **保护隐私** - 保护举报者和被举报者的隐私
* **寻求建议** - 必要时咨询其他维护者或专家

#### ⚖️ 处理措施
根据违规严重程度，可能采取的措施包括：

1. **口头警告** - 私下提醒注意行为规范
2. **书面警告** - 正式警告并记录在案
3. **临时限制** - 暂时限制参与项目活动
4. **永久封禁** - 从项目社区中永久移除
5. **法律行动** - 对于严重违法行为，可能采取法律措施

#### 📋 后续跟进
* **监督改进** - 跟踪违规者的后续行为
* **社区通知** - 必要时向社区通报处理结果
* **政策更新** - 根据案例完善行为准则
* **预防措施** - 采取措施防止类似事件再次发生
